# 聚水潭入库网站 — 部署到 Streamlit Community Cloud 完整指南

> 部署一次, 同事打开浏览器就能用; 你改完代码 push 一下 GitHub, 30 秒后自动生效

---

## 一次性准备 (大约 20 分钟)

### 0. 这套文件包含什么

```
聚水潭网站/
├── app.py                # Streamlit 界面 — 主程序
├── factories.py          # 4 家工厂账单解析
├── jushuitan_import.py   # 聚水潭入库 Excel 生成
├── requirements.txt      # Python 依赖清单 (Streamlit Cloud 会自动装)
├── .gitignore            # Git 忽略文件 (xlsx / 凭证不要 push)
└── 部署指南.md            # 就是这份
```

---

### 1. 注册 GitHub 账号 (如果还没有)

打开 https://github.com → Sign up → 填邮箱密码用户名 → 验邮箱即可。

---

### 2. 装 GitHub Desktop (Mac 图形客户端)

下载 https://desktop.github.com/ → 安装 → 用刚注册的账号登录。

> 这步是为了你后续改代码不用敲命令, 鼠标点点就能 push。

---

### 3. 用 GitHub Desktop 新建一个仓库 + 把这套文件放进去

1. 打开 GitHub Desktop → **File → New Repository**
2. **Name** 填: `jushuitan-import` (随便取, 全英文小写)
3. **Local Path** 选你 Mac 桌面的某个位置 (会创建一个 jushuitan-import 文件夹)
4. ✅ 勾选 "Initialize this repository with a README"
5. 点 **Create Repository**

然后:
6. 找到刚才创建的 `~/Desktop/jushuitan-import/` 这个文件夹
7. 把本网站文件夹里的 **app.py / factories.py / jushuitan_import.py / requirements.txt / .gitignore** 这 5 个文件**复制进去** (跟那个 README.md 同级)
8. 回到 GitHub Desktop, 它会自动检测到这 5 个新文件
9. 左下方填 **Summary**: `初始版本`, 点 **Commit to main**
10. 右上方点 **Publish repository** → 弹窗里**取消勾选** "Keep this code private" (公开仓库才能用 Streamlit Cloud 免费版) → 点 **Publish repository**

> ⚠️ 注意: 公开仓库 = 任何人能看到你的代码 (但**不会看到客户上传的工厂账单**, 那是运行时临时数据)。如果代码里含敏感信息 (比如客户名单), 请保持 Private — 这种情况下后面 Streamlit Cloud 也能连私有仓库, 只是要付 $20/月。

完成后浏览器打开 `https://github.com/你的用户名/jushuitan-import` 应该能看到那些文件。

---

### 4. 给 Streamlit 账号开两步验证 (必做, 接飞书前置)

**为什么必做**: 飞书 App Secret 会存在 Streamlit Cloud 后台。如果你 Streamlit 账号密码被泄露, 攻击者能进 dashboard 看到密钥。两步验证 = 密码 + 手机验证, 极大降低被盗风险。

**操作** (注册 Streamlit 后):
1. 登录 https://share.streamlit.io
2. 右上角头像 → **Account settings** → **Security**
3. 开启 **Two-factor authentication** (扫码绑定 Google Authenticator / 微软 Authenticator / 1Password)
4. 备份 recovery codes (打印或抄下来, 别拍照存手机相册)

---

### 5. 注册 Streamlit Community Cloud 并部署

1. 打开 https://streamlit.io/cloud
2. 点 **Sign up** → 选 **Continue with GitHub** → 授权
3. 进入 dashboard 后点 **New app** (右上)
4. 表单填:
   - **Repository**: `你的用户名/jushuitan-import`
   - **Branch**: `main`
   - **Main file path**: `app.py`
   - **App URL** (可选): 比如 `jushuitan-import` → 最终网址会是 `https://jushuitan-import.streamlit.app`
5. 点 **Deploy!**

等 2-3 分钟, 状态从 "Building..." → "Running" 时就 OK 了。

⚠️ **首次打开网页会看到「飞书连接失败」**, 因为还没配凭证, 下一步搞。

---

### 6. 配置飞书密钥 (Streamlit Secrets)

**这步是关键 — 凭证不在 GitHub 仓库里, 而是放 Streamlit 后台加密存。**

1. 回到 Streamlit Cloud 你那个 App 的页面
2. 右下角 **⋮ 三个点** → **Settings** → 左侧 **Secrets**
3. 输入框里粘贴 (TOML 格式, 注意 = 两边的引号):

```toml
FEISHU_APP_ID = "cli_xxxxxxxxxx"
FEISHU_APP_SECRET = "你的真实 secret"
```

4. 点 **Save**

App 会自动重启, 30 秒后刷新网页, 顶部应该显示「✓ 飞书已连接」。

> ⚠️ **不要把这两行写进代码 / requirements / 任何 .py 文件**! 那样会 push 到 GitHub 公开仓库立刻泄漏。Streamlit Secrets 才是正确的存放位置。

> ✅ 在哪里看你的 APP_ID / APP_SECRET: 飞书开放平台 → 你的应用 → 凭证与基础信息 → "App Secret" 那栏点显示。

---

### 7. (可选但建议) 把仓库改成 Private

公开仓库代码任何人可见, 虽然密钥不在代码里, 但工厂 / 客户名相关的业务逻辑可能不想让外人看。

Streamlit Cloud 免费版**也支持 Private 仓库**, 只是需要在 GitHub 授权时勾上"私有仓库访问权限":

1. GitHub → 你的仓库 → **Settings** → 拉到底 **Danger Zone** → **Change visibility** → **Make private**
2. Streamlit 那边 App 设置里重新授权 GitHub (可能要求重新登录)

---

**网址 (告诉同事的就是这个)**:
```
https://你的-app-名.streamlit.app
```

## 日常: 改完代码怎么发布

你在 Mac 上改 `app.py` 或 `factories.py`:

1. 打开 GitHub Desktop, 它会自动检测到改动
2. 左下方填 **Summary** (写一句改了什么)
3. 点 **Commit to main**
4. 点上方 **Push origin**

完事。Streamlit Cloud 会在大约 30 秒后自动重新部署, 同事刷新网页就是新版本。

## 常见问题

### 同事访问慢 / 转圈
首次访问会冷启动 (10-30 秒), 之后正常。如果一直慢, 可能 Streamlit Cloud 在国内被限速, 后续可考虑迁阿里云函数计算。

### 上传文件大小有限制吗
Streamlit 默认上传上限 200MB, 工厂账单一般几百 KB, 完全够。

### 处理超时吗
Streamlit Cloud 没有硬性超时, 但单 app 内存上限 1GB。当前处理量秒级搞定, 不用担心。

### 多个同事同时用会冲突吗
不会。每个浏览器会话独立, 文件互不影响。

### .xls 文件能处理吗
**不能**, Streamlit Cloud 上没有 libreoffice。需要同事先在 Excel/WPS 里另存为 .xlsx。

### 想加新功能
跟 evo 说要加什么, 我改完 push, 同事刷新就行。
